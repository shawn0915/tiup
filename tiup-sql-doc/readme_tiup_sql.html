TiUP Component

数据库连接与管理
还能更简单？

tiup sql 为 TiUP 带来通用 SQL 客户端能力。一条命令连接 MySQL 或 TiDB。批量执行。多格式导出。安全凭证管理。

查看命令示例 测试报告

复杂的数据库工具让人疲惫？

tiup sql 采用模块化架构，将连接、执行、格式化、日志完全解耦。每个模块只做一件事，组合起来覆盖全部场景。

CONNECT

连接管理

DSN 解析、TLS 加密、Playground 自动发现、连接配置持久化。支持 URL、键值对、环境变量三种传参方式。

BATCH

批量执行

SQL 文件、目录批量执行、stdin 管道、heredoc。引号感知分割器正确处理字符串内的分号。

REPL

交互式会话

交互式 REPL 支持元命令、多行编辑、输出格式实时切换。非 TTY 环境自动降级为批量模式。

FORMAT

多格式输出

表格、CSV、JSON（含 pretty / rows）、TSV、垂直表格。支持无头模式与计时显示。

LOG

查询日志

可选记录所有执行的 SQL、耗时、影响行数。自动脱敏 DSN 中的密码字段。

SECURITY

安全合规

密码交互式输入、TLS 默认验证、SQL 注入防护、配置文件 0o600 权限。无明文密码泄露风险。

如何快速连接数据库？

无论本地测试还是生产环境，一行命令即可建立连接。支持 5 种连接方式，总有一种适合你。

1

URL 格式

最直观的方式。用户名、密码、主机、端口、数据库一次性写入。

tiup sql mysql://root:password@192.168.1.101:3308/test_db
2

键值对参数

适合脚本和 CI/CD 场景。参数清晰，易于维护。

tiup sql --host 192.168.1.101 --port 3308 --user root test_db
3

保存的连接配置

常用连接保存一次，后续通过别名直接调用。告别冗长的连接字符串。

# 保存 tiup sql --save-connection prod mysql://root@10.0.1.100:3306/app_db # 使用 tiup sql -c prod -e "SELECT 1"
4

环境变量传密

密码通过 TIUP_SQL_PASSWORD 环境变量传入。命令历史中不会出现敏感信息。

export TIUP_SQL_PASSWORD=Shawn2026 tiup sql -e "SHOW DATABASES" mysql://root@192.168.1.101:3308/
5

Playground 自动发现

本地已启动 playground？无需手动输入地址，自动扫描并连接。

tiup sql --playground

查询结果如何优雅导出？

7 种输出格式覆盖从人工查看到程序解析的全部场景。一条命令切换格式。

TABLE

默认表格

适合终端直接查看。ASCII 表格对齐，自动计算列宽，支持中文。

CSV

逗号分隔

Excel 直接打开。可搭配 --no-header 输出纯数据。

JSON

JSON 数组

API 友好。可选 json-pretty 美化或 json-rows NDJSON。

TSV

制表符分隔

Unix 管道最佳伴侣。与 awk、cut、sort 无缝协作。

VERTICAL

垂直展示

宽表救星。类似 MySQL \G，每行数据纵向排列。

敏感凭证如何安全处理？

从输入到存储到日志，全链路保护密码与证书。不泄露、不残留、不裸奔。

1

交互式密码

-p 为 bool flag。用户必须交互输入密码，不在命令历史中留存。

2

环境变量隔离

支持 TIUP_SQL_PASSWORD。进程退出后环境变量自动清除。

3

日志自动脱敏

SanitizedDSN 将密码替换为 ***。日志文件永不出现明文。

4

TLS 默认验证

证书验证默认开启。--tls-skip-verify 必须显式声明并附带警告。

5

SQL 注入防护

元命令使用 isValidIdentifier 校验。非法字符直接拒绝。

6

配置文件权限

connections.yaml 创建时权限设为 0o600。仅所有者可读写。

现在就开始？

构建

2

连接

export TIUP_SQL_PASSWORD=your_password tiup-sql -e "SHOW DATABASES" mysql://root@127.0.0.1:3306/
3

验证

go test ./components/sql/... -v
